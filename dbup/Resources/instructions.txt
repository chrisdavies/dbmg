dbup -c <connection-string> [-f <path-to-sql>] [-t <versioning-table>] 
     [-i <initial-file> | -a <after-file>] [-p <db-provider>]

Options:
-c	specifies the connection string of the database to connect to
	defaults to the "default" connection string in dbup.config

-f	the path to the directory containing the sql files to use in the migration
	defaults to "./db"

-t	the name of the table to create and use for tracking db version info
	defaults to "dbup"

-i	the initial file overrides versioning and runs the file specified by
	<initial-file> and all subsequent files

-a	run all files whose name is greater than <after-file>

-p	the database provider
	defaults to "postgres"

For more information, see: https://github.com/chrisdavies/dbup
